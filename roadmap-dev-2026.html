Roadmap Dev 2026

Frontend · Backend · Fullstack

Do absoluto zero ao profissional sênior.
React · React Native · Node.js · TypeScript · Cloud · IA

🖥 Frontend Dev ⚙️ Backend Dev ⚡ Fullstack Dev
Intro Frontend Backend Fullstack Ferramentas Carreira
📌 Como usar este roadmap

Este documento é um guia de rota, não uma lista de obrigações simultâneas. Siga a ordem proposta. Pule uma fase só quando estiver confortável praticando, não apenas depois de ler. Cada fase tem projetos reais — faça todos.

Tempo total estimado: Frontend ~12 meses · Backend ~14 meses · Fullstack ~20–24 meses (dedicação diária de 2–4h).

⚠️ JavaScript antes de TypeScript

TypeScript não substitui JavaScript. Ele é JavaScript com superpoderes de tipagem. Para entender TS profundamente você precisa dominar os fundamentos de JS primeiro: closures, protótipos, event loop, hoisting, escopo, assincronicidade. O roadmap respeita essa ordem.

💡 Princípio base

Consistência > Intensidade. Estudar 2 horas todos os dias por 12 meses é infinitamente mais eficaz que estudar 12 horas por 2 semanas e parar. Projetos práticos valem mais que 10 cursos assistidos passivamente.

Por que começar aqui?

HTML e CSS são a fundação de tudo na web. Sem entender estrutura semântica e layout, você vai sofrer em qualquer framework depois.

🏗 HTML Semântico ~2 semanas
Tópicos

HTML semântico melhora acessibilidade, SEO e manutenibilidade. Leitores de tela e indexadores dependem da estrutura correta.

⚡ Fastify — Performance Moderna ~2 semanas
Por que Fastify?
  • 2–3× mais rápido que Express
  • Schema validation nativo com JSON Schema
  • Plugin system (encapsulamento)
  • TypeScript first
  • Logging com Pino integrado
Quando usar Express vs Fastify
Express: projetos legados, ecossistema maduro, aprendizado.
Fastify: novos projetos, alta performance, TypeScript nativo, microserviços.

Trilha 03

Fullstack Developer

Frontend + Backend + Mobile + Cloud + IA + DevOps + Produto · ~20–24 meses

📋 Estrutura da trilha Fullstack

A trilha Fullstack inclui tudo das trilhas Frontend e Backend. As fases abaixo cobrem os tópicos exclusivos do Fullstack: integração, mobile avançado, cloud, IA, IoT e produto. Complete as trilhas FE e BE antes ou em paralelo.

+
🔗 Arquitetura Full-Stack Moderna ~4–5 semanas
Integração

  • Compartilhar tipos TS entre FE e BE
  • CORS configurado para produção
  • Environment variables em todos os ambientes
  • API client centralizado (Axios instance)
  • tRPC — type-safe API sem código gerado
Deploy Completo
  • Frontend: Vercel
  • Backend: Railway ou Render
  • Banco: Neon (PostgreSQL serverless)
  • Storage: Cloudflare R2 ou S3
  • Domínio customizado com SSL
  • Preview deploys por branch

IA para devs em 2026

Você não precisa ser cientista de dados para integrar IA. Em 2026, developers que sabem consumir e orquestrar LLMs têm um diferencial enorme. Foco em integração prática, não em treinamento de modelos.

🧠 Conceitos Fundamentais de IA ~2 semanas
📊
Machine Learning
Algoritmos que aprendem padrões a partir de dados. Supervisionado, não-supervisionado, por reforço.
🔮
Deep Learning
Redes neurais profundas. Base para modelos modernos de linguagem e visão computacional.
💬
LLMs
Large Language Models. GPT-4, Claude, Gemini, Llama. Modelos treinados em texto para geração de linguagem.
🔢
Tokens e Embeddings
Tokens são unidades de texto. Embeddings são representações vetoriais de semântica — base do RAG.
🔍
RAG
Retrieval-Augmented Generation. Busca documentos relevantes e injeta no contexto do LLM. Potente para chatbots empresariais.
🤖
Agentes de IA
LLMs que planejam e executam ações: buscar na web, chamar APIs, escrever e rodar código.
🖼
IA Multimodal
Modelos que processam texto, imagem, áudio e vídeo simultaneamente. GPT-4o, Gemini 2.0.
📐
Prompt Engineering
Arte de estruturar prompts. System prompts, few-shot, chain-of-thought, structured outputs.

  • OpenAI API — chat, completions, embeddings, vision
  • Anthropic API (Claude) — streaming, tools
  • Gemini API — multimodal
  • Streaming de respostas para o frontend
  • Function Calling / Tool Use
  • Structured Outputs com JSON Schema

RAG e Vector DB
  • pgvector no PostgreSQL

  • Chunking estratégico de documentos
  • Embeddings com text-embedding-3
  • Hybrid search: vetorial + full-text
  • Reranking para qualidade
LangChain e Agentes
  • LangChain.js — chains e memory
  • LangGraph — agentes com estado
  • Tools: web search, calculator, code
  • Multi-agent systems
  • Mastra.ai — agentes em TypeScript
Local e Open Source
  • Ollama — LLMs locais
  • LM Studio — GUI local
  • Llama 3, Mistral, Qwen localmente
  • Hugging Face — modelos e spaces
  • TensorFlow.js — ML no browser

Evolução de Carreira

Do primeiro commit ao primeiro emprego e além

💡 Portfólio que contrata

3 projetos bons > 20 projetos mediocres. Cada projeto do portfólio deve ter: README profissional, deploy funcionando, código limpo, descrição do problema que resolve, tecnologias usadas e suas escolhas arquiteturais.

📁Portfólio por Nível
Júnior (primeiros 6–12 meses)
  • Portfólio web com Next.js + deploy
  • 3 projetos com README completo
  • Contribuição em repo open source
  • GitHub ativo (commits frequentes)
Pleno (1–3 anos)
  • SaaS em produção com usuários
  • API com documentação OpenAPI
  • App mobile publicado
  • Blog técnico ou posts no LinkedIn
Sênior (3+ anos)
  • Projetos em escala real
  • Open source relevante
  • Mentoria ou artigos técnicos
  • Palestras em meetups/eventos
💼Como Conseguir o Primeiro Emprego
  • LinkedIn otimizado com projetos
  • GitHub com README profile
  • Currículo de 1 página, focado em resultados
  • Networking ativo em comunidades Discord
  • Participar de hackathons
  • Freela para ganhar experiência
  • Trabalho voluntário em ONGs tech
  • Candidatura ativa: 5–10 vagas por dia

Aprenda em público. Documente sua jornada no LinkedIn, Twitter/X ou blog. Mostre o que você está aprendendo, os erros que cometeu, como resolveu. Isso constrói reputação antes do emprego, atrai oportunidades e reforça o aprendizado.
